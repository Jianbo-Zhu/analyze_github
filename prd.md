# GitHub开源项目分析系统

## 项目概述

本项目旨在全面分析GitHub平台上2025年以来最活跃且影响力较大的开源项目，通过多维度数据分析，揭示开源生态系统的发展趋势、技术偏好和贡献者特征，为开发者、研究人员和技术决策者提供有价值的洞察。

## 功能需求

### 1. 数据采集模块

#### 1.1 项目筛选
- 采集GitHub 2025年以来提交次数超过50次的项目
- 筛选星标数超过1000且Fork数超过100的项目
- 获取前5000个符合条件的项目

#### 1.2 数据获取
- 使用PyGithub库与GitHub API交互
- 采集项目基本信息：名称、描述、创建日期、最后更新日期
- 采集技术栈信息：主要编程语言、依赖关系
- 采集贡献者信息：用户名、邮箱、提交历史、贡献频率
- 采集项目统计数据：星标数、Fork数、issue数量、PR数量

#### 1.3 数据存储
- 将原始数据存入MySQL数据库
- 设计合理的数据模型和表结构
- 实现数据增量更新机制

### 2. 数据分析模块

#### 2.1 编程语言分布分析
- 统计各类编程语言在TOP项目中的占比
- 分析语言使用趋势变化
- 识别新兴编程语言和衰退编程语言

#### 2.2 贡献者画像分析
- 地域分布：通过贡献者IP或资料分析地理分布
- 性别分布：通过用户名和个人资料进行合理推测
- 年龄分析：基于账号创建时间和活跃周期推测
- 贡献活跃度：分析贡献频率、提交质量和持续时间

#### 2.3 项目领域分类分析
- 基于项目描述和关键词进行领域分类
- 分析各领域项目数量和影响力分布
- 识别热门领域和新兴研究方向

#### 2.4 补充分析维度
- 项目生命周期分析：各阶段项目数量和特征
- 社区健康度评估：issue解决率、PR合并速度、讨论活跃度
- 技术栈组合分析：主流技术栈搭配和依赖关系
- 开源许可证分布：不同许可证类型的使用情况
- 项目活跃度时间模式：工作日vs周末、高峰期分析

### 3. 可视化报告模块

#### 3.1 PDF报告生成
- 生成结构化的PDF分析报告
- 包含详细的数据表格和静态图表
- 提供报告摘要和关键发现

#### 3.2 交互式HTML报告
- 开发现代化的HTML报告界面
- 集成GSAP动画框架实现流畅的交互效果
- 使用SVG绘制动态、交互式图表
- 实现数据筛选和下钻分析功能
- 支持响应式设计，适配不同设备

## 技术架构

### 技术栈
- **开发语言**：Python 3.12
- **数据采集**：PyGithub库
- **数据存储**：MySQL 8.0+
- **数据分析**：Pandas, NumPy, Scikit-learn
- **可视化**：Matplotlib, Seaborn, Plotly
- **Web前端**：HTML5, CSS3, JavaScript
- **动画框架**：GSAP (GreenSock Animation Platform)
- **容器化**：Docker, Docker Compose

### 系统架构
1. **数据采集层**：负责从GitHub API获取原始数据
2. **数据存储层**：MySQL数据库存储采集和处理后的数据
3. **数据处理层**：Python分析脚本进行数据清洗和分析
4. **报告生成层**：生成PDF和HTML格式的分析报告
5. **部署层**：Docker容器化部署，确保环境一致性

## 数据模型

### 核心数据表
1. **projects**：项目基本信息
2. **languages**：项目语言分布
3. **contributors**：贡献者信息
4. **commits**：提交记录
5. **statistics**：项目统计数据
6. **topics**：项目主题标签

## 风险评估

1. **API限流风险**：GitHub API有请求频率限制，需实现请求节流和缓存机制
2. **数据完整性风险**：部分贡献者信息可能不完整，影响画像分析准确性
3. **性能挑战**：处理大量数据可能面临性能瓶颈，需优化查询和计算
4. **准确性风险**：性别和年龄等信息基于推测，存在一定误差
